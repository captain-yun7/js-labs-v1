주요 DOM 선택자 메서드

메서드 반환 타입 라이브 업데이트 특징
getElementById() Element 객체 N/A (단일 요소) 단일 요소 반환, ID가 없으면 null 반환
getElementsByClassName() HTMLCollection O (라이브) DOM이 변경되면 자동으로 업데이트됨
getElementsByTagName() HTMLCollection O (라이브) DOM이 변경되면 자동으로 업데이트됨
querySelector() Element 객체 N/A (단일 요소) 첫 번째 일치 요소만 반환, 없으면 null
querySelectorAll() NodeList X (정적) DOM이 변경되어도 결과 업데이트 안됨

선택자 결론

언제 어떤 선택자를 사용해야 할까요?

1. getElementById()

  • 가장 빠른 성능
  • ID가 있는 특정 요소를 선택할 때 최적
  • ID가 없는 요소에는 사용 불가

2. getElementsByClassName() / getElementsByTagName()

  • DOM 변경을 자동 반영하는 라이브 컬렉션
  • 단순한 클래스나 태그 선택에 효율적
  • DOM이 자주 변경되는 상황에서 유용

3. querySelector() / querySelectorAll()

  • 다양한 CSS 선택자 지원
  • 복잡한 선택 조건을 간결하게 표현 가능
  • 다른 방법보다 약간 느림
  • 복잡한 선택이 필요하거나 정적 스냅샷이 필요할 때 사용
